# База данных и миграции

## 1. Основные таблицы

В проекте используются следующие ключевые таблицы:

- `sources` — источники новостей (RSS/site), расписание, языковые настройки.
- `articles_raw` — сырые статьи после парсинга и дедупликации.
- `articles_draft` — переведённые черновики для редактора.
- `llm_tasks` — выполненные/выполняемые LLM-задачи.
- `llm_presets` — редактируемые пресеты LLM (добавлено в итерации 4).
- `publications` — публикации в каналах.
- `moderation_rules` — правила модерации.
- `users` — пользователи (админ/редактор).

## 2. История миграций

### `20260325_0001_init_schema`
- Создаёт базовую схему MVP.
- Включает таблицы: `sources`, `articles_raw`, `articles_draft`, `llm_tasks`, `publications`, `moderation_rules`, `users`.

### `20260325_0002_add_llm_presets`
- Добавляет таблицу `llm_presets`.
- Сидит базовые пресеты:
  - `summary`
  - `rewrite_style`
  - `title_hashtags`
- Нормализует значения `articles_draft.status`.
- Добавляет check-constraint для допустимых статусов.

## 3. Запуск миграций

Обычный запуск:

```bash
alembic upgrade head
```

Генерация SQL (без выполнения):

```bash
alembic upgrade head --sql
```

## 4. Важные замечания

- Основная целевая БД: PostgreSQL.
- Для smoke-check в SQLite в миграции `20260325_0002` предусмотрена совместимая ветка.
- Рекомендуется применять миграции только через Alembic, а `scripts/init_db.py` использовать как fallback для локальной инициализации.
